Step 1: Prepare Your Mac Mini

If you're starting with a fresh Mac Mini, run through the macOS setup wizard normally. Once you're at the desktop, do these housekeeping steps:

Enable Remote Access

Open System Settings ‚Üí General ‚Üí Sharing and enable:

  • Remote Login (SSH) ‚Äî So you can manage the Mac Mini from another computer
  • Screen Sharing (optional) ‚Äî For remote desktop access when needed

Prevent Sleep

Your Mac Mini needs to stay awake 24/7 for OpenClaw to run continuously:

  • Go to System Settings ‚Üí Energy
  • Set "Turn display off after" to your preference (this only affects the display, not the computer)
  • Enable "Prevent automatic sleeping when the display is off"
  • Enable "Start up automatically after a power failure"

Install Homebrew

Open Terminal (Applications ‚Üí Utilities ‚Üí Terminal) and install Homebrew, the macOS package manager:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Follow the on-screen instructions. After installation, you may need to add Homebrew to your PATH as instructed.

Step 2: Install Node.js

OpenClaw requires Node.js version 22 or later. The easiest way to install it on macOS is with Homebrew:

brew install node@22

Verify the installation:

node --version
# Should output v22.x.x or higher

If you prefer version management, you can use nvm or fnm instead ‚Äî but for a dedicated OpenClaw machine, the Homebrew install is simplest.

Step 3: Install OpenClaw

Now for the main event. Install OpenClaw globally via npm:

npm install -g openclaw@latest

Verify it's installed:

openclaw --version

You should see the version number printed. That's it ‚Äî OpenClaw is installed on your Mac Mini.

Step 4: Run the Onboarding Wizard

OpenClaw includes a guided setup wizard that handles configuration, API keys, and daemon installation. Run it with:

openclaw onboard --install-daemon

The wizard will walk you through:

  1. AI model provider ‚Äî Choose Anthropic (Claude), OpenAI (GPT-4), or another provider
  2. API key ‚Äî Paste your API key from the provider's dashboard
  3. Default model ‚Äî Pick which model to use (e.g., Claude Sonnet, GPT-4)
  4. Agent name and persona ‚Äî Give your AI assistant an identity
  5. Daemon installation ‚Äî Sets up a launchd service so OpenClaw starts automatically on boot

The --install-daemon flag is crucial for your openclaw mac mini setup. It creates a macOS launchd plist that ensures OpenClaw starts when the Mac Mini boots and restarts automatically if it ever crashes. This is what makes your setup truly "always-on."

Step 5: Connect Your Messaging Channels

With the Gateway running, it's time to connect the messaging apps where you want to interact with your AI assistant. OpenClaw supports 10+ platforms ‚Äî here are the most common:

WhatsApp

openclaw channels login whatsapp

This displays a QR code in your terminal. Scan it with WhatsApp on your phone (Settings ‚Üí Linked Devices ‚Üí Link a Device). Your AI assistant is now reachable via WhatsApp.

Slack

openclaw channels login slack

This opens a browser flow to create a Slack app in your workspace. Follow the prompts to authorize OpenClaw. Once connected, you can message your AI assistant in any Slack channel or DM.

Discord

openclaw channels login discord

You'll create a Discord bot and paste the token. The wizard guides you through the Discord Developer Portal setup.

Telegram

openclaw channels login telegram

Create a bot via BotFather on Telegram and paste the token. Simple and fast.

You can connect multiple channels simultaneously. Your OpenClaw instance handles all of them from a single Gateway process.

Step 6: Access the Web Control UI

OpenClaw provides a browser-based dashboard for chatting with your agent, managing configuration, and monitoring sessions. By default, it's available at:

http://localhost:3000

On your Mac Mini, open Safari or Chrome and navigate to that URL. You'll see the OpenClaw Web UI where you can:

  • Chat with your AI assistant directly
  • View and manage active sessions
  • Configure skills, cron jobs, and automations
  • Monitor Gateway health and logs

Remote Access with Tailscale

Want to access your Mac Mini's OpenClaw from your phone, laptop, or anywhere in the world? Install Tailscale (free for personal use):

Sign in, install Tailscale on your other devices, and you can access the Web UI via your Mac Mini's Tailscale IP ‚Äî no port forwarding, no exposed ports, fully encrypted.

Step 7: Configure Your Workspace

Now that OpenClaw is running, customize it for your needs. The workspace is where you define your agent's personality, skills, and automations.

The DNA Files

OpenClaw uses a set of "DNA files" ‚Äî markdown files that define how your agent behaves:

  • SOUL.md ‚Äî Your agent's identity, role, and behavior guidelines
  • AGENTS.md ‚Äî Workspace rules, channel routing, and operational procedures
  • TOOLS.md ‚Äî Environment-specific notes (SSH hosts, device names, etc.)

Edit these files to shape your AI assistant's personality and capabilities. For example, you might tell it to act as a customer support agent, a research assistant, or a personal chief of staff.

Adding Skills

Skills are custom capabilities you add to your agent. They can range from simple text templates to complex automations that call APIs, read files, and control browsers. Check the OpenClaw documentation for the skill format, or browse community skills on ClawHub.

Setting Up Cron Jobs

Want your AI assistant to run tasks on a schedule? OpenClaw supports cron-style scheduling. For example, you could set up a morning briefing that runs at 7 AM every day, compiling your emails, calendar, and news into a summary delivered to your WhatsApp.

Optimizing Your Mac Mini for 24/7 Operation

Your openclaw mac mini setup is functional now, but here are pro tips for rock-solid 24/7 operation:

Use Ethernet Over Wi-Fi

Wi-Fi can drop or lag. For a server that runs 24/7, a wired Ethernet connection eliminates connectivity issues. The Mac Mini has a built-in Gigabit Ethernet port (or 10Gb on higher-end models).

Set a Static IP or Use Tailscale

Assign a static IP on your local network (via your router's DHCP settings) so your Mac Mini is always reachable at the same address. Or use Tailscale for a stable hostname regardless of network changes.

Enable Automatic Login

Go to System Settings ‚Üí Users & Groups ‚Üí Automatic Login and set your user account. This ensures the Mac Mini is ready after a power outage without needing someone to type a password.

Monitor with the OpenClaw Dashboard

The Web Control UI shows Gateway health, connected channels, and session activity. Bookmark it on your phone for quick check-ins.

Keep macOS Updated (Carefully)

Apply security updates regularly, but avoid major macOS upgrades (e.g., Sonoma ‚Üí Sequoia) without checking OpenClaw compatibility first. Minor updates are generally safe.

Mac Mini vs. VPS: Which Is Right for You?

Wondering whether to run OpenClaw on your Mac Mini or a cloud VPS? Here's the honest comparison:

  • Choose Mac Mini if: You want zero monthly hosting costs, need macOS-specific features (iMessage, Apple Shortcuts), prefer physical control over your hardware, or already own one.
  • Choose a VPS if: You need guaranteed uptime with data center redundancy, want to avoid managing hardware, need a static public IP, or are setting up for a distributed team.

Many power users run both ‚Äî a Mac Mini at home for personal use and a VPS for production deployments. OpenClaw works identically on both.

Troubleshooting Common Issues

OpenClaw Won't Start After Reboot

Check that the launchd daemon is loaded:

launchctl list | grep openclaw

If it's not listed, reload the plist:

launchctl load ~/Library/LaunchAgents/com.openclaw.gateway.plist

WhatsApp QR Code Won't Scan

Make sure your terminal window is large enough to display the full QR code. Try increasing the terminal font size or zooming out. If the code expires, run the login command again.

Can't Access Web UI Remotely

If you're on the same network, try using the Mac Mini's local IP (e.g., http://192.168.1.x:3000) instead of localhost. For remote access, set up Tailscale as described above.

High CPU Usage

This is rare with OpenClaw itself ‚Äî it's typically idle between messages. If you see sustained high CPU, check if a skill or cron job is running a heavy operation. The Web UI's session view can help identify the culprit.
