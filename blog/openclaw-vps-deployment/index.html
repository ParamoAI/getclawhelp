What VPS Specs Do You Need?

OpenClaw is a Node.js application ‚Äî it's lightweight. Here are the minimum and recommended specs:

  • Minimum: 1 vCPU, 1GB RAM, 20GB SSD ‚Äî Works for personal use with light usage
  • Recommended: 2 vCPU, 2‚Äì4GB RAM, 40GB SSD ‚Äî Handles business workloads, multiple channels, and browser automation
  • Heavy use: 4 vCPU, 8GB RAM ‚Äî For running local AI models alongside OpenClaw, or high-volume multi-agent setups

For a typical small business deployment, the recommended tier is perfect. OpenClaw idles at very low CPU usage and spikes briefly when processing messages.

Deploy on Hetzner (Best Value)

Hetzner offers the most compute per dollar, making it our top pick for openclaw vps deployment.

Step 1: Create Your Server

  1. Sign up at hetzner.com/cloud
  2. Click "Add Server"
  3. Choose your location (Ashburn for US, Falkenstein for EU)
  4. Select Ubuntu 24.04 as the image
  5. Choose CX22 (2 vCPU, 4GB RAM, 40GB SSD) ‚Äî ‚Ç¨4.85/month
  6. Add your SSH key (or set a root password)
  7. Click "Create & Buy Now"

Your server will be ready in about 30 seconds. Copy the IP address from the dashboard.

Step 2: Connect and Secure

# SSH into your new server
ssh root@YOUR_SERVER_IP

# Update the system
apt update && apt upgrade -y

# Create a non-root user
adduser openclaw
usermod -aG sudo openclaw

# Set up firewall
ufw allow OpenSSH
ufw allow 3000/tcp
ufw enable

# Switch to the new user
su - openclaw

Step 3: Install Node.js and OpenClaw

# Install Node.js 22 via NodeSource
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs

# Verify
node --version  # Should show v22.x.x

# Install OpenClaw
sudo npm install -g openclaw@latest

# Run the onboarding wizard with daemon
openclaw onboard --install-daemon

The wizard guides you through API key setup, model selection, and daemon installation. The systemd daemon ensures OpenClaw starts on boot and restarts on crashes.

Step 4: Connect Channels

# Connect WhatsApp
openclaw channels login whatsapp

# Connect Slack
openclaw channels login slack

# Connect any other channels you need
openclaw channels login telegram

That's it. Your openclaw vps deployment on Hetzner is complete. Access the Web Control UI at http://YOUR_SERVER_IP:3000.

Deploy on DigitalOcean (Easiest)

DigitalOcean's clean interface makes it the friendliest option for first-time VPS users.

Step 1: Create a Droplet

  1. Sign up at digitalocean.com
  2. Click "Create ‚Üí Droplets"
  3. Choose a region (New York or San Francisco for US)
  4. Select Ubuntu 24.04
  5. Choose Basic ‚Üí Regular ‚Äî $12/month (1 vCPU, 2GB RAM, 50GB SSD)
  6. Add your SSH key
  7. Click "Create Droplet"

Step 2: Secure and Install

usermod -aG sudo openclaw
ufw allow OpenSSH
ufw allow 3000/tcp

# Install Node.js

sudo apt install -y nodejs

# Install and set up OpenClaw
sudo npm install -g openclaw@latest

Step 3: Connect Channels and Go

openclaw channels login whatsapp
openclaw channels login slack

DigitalOcean also offers managed databases, load balancers, and monitoring if you want to build a more robust infrastructure around your OpenClaw deployment later.

Deploy on AWS Lightsail

If you're already using AWS services, Lightsail keeps everything under one roof with predictable pricing.

Step 1: Create a Lightsail Instance

  1. Go to lightsail.aws.amazon.com
  2. Click "Create instance"
  3. Select your region (e.g., us-east-1)
  4. Choose Linux/Unix ‚Üí Ubuntu 24.04
  5. Select the $10/month plan (2 vCPU, 2GB RAM, 60GB SSD)
  6. Name your instance and click "Create"

Step 2: Configure Networking

In the Lightsail console, go to your instance's Networking tab and add a firewall rule:

  • Application: Custom
  • Protocol: TCP
  • Port: 3000

This allows access to the OpenClaw Web Control UI.

Step 3: SSH In and Install

Click the terminal icon in the Lightsail console (or SSH with your key):

# Update system
sudo apt update && sudo apt upgrade -y

# Install Node.js
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs

# Install OpenClaw
sudo npm install -g openclaw@latest
openclaw onboard --install-daemon

# Connect channels
openclaw channels login whatsapp
openclaw channels login slack

Lightsail also offers static IPs (free when attached to an instance) and easy DNS management if you want to put a domain name on your OpenClaw Web UI.

Post-Deployment: Essential Configuration

Regardless of which provider you chose, here are the important steps to finalize your openclaw vps deployment:

Set Up HTTPS (Optional but Recommended)

If you want to access the Web UI securely over HTTPS with a custom domain:

# Install Nginx and Certbot
sudo apt install -y nginx certbot python3-certbot-nginx

# Configure Nginx as a reverse proxy (create /etc/nginx/sites-available/openclaw)
# Point your domain's DNS to your server's IP
# Run Certbot for free SSL
sudo certbot --nginx -d your-domain.com

Set Up Remote Access with Tailscale

Alternatively, skip HTTPS and use Tailscale for secure private access:

curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up

Then access the Web UI via your server's Tailscale IP. No ports exposed to the public internet, no SSL certificates to manage.

Enable Automatic Updates

Keep your server's OS packages updated automatically:

sudo apt install -y unattended-upgrades
sudo dpkg-reconfigure -plow unattended-upgrades

Set Up Backups

All three providers offer automated backups:

  • Hetzner: Enable backups for +20% of server cost (‚Ç¨0.97/month for CX22)
  • DigitalOcean: Enable weekly backups for +20% of droplet cost
  • AWS Lightsail: Automatic snapshots are included in the plan

Configure Your Workspace

Customize your OpenClaw agent by editing the workspace DNA files:

  • SOUL.md ‚Äî Define your agent's identity and behavior
  • AGENTS.md ‚Äî Set up channel routing and operational rules
  • TOOLS.md ‚Äî Add environment-specific configurations

  • Docker ‚Äî Better for servers running multiple services, easier updates and rollbacks

Security Hardening

Your VPS is on the public internet, so basic security hygiene is important:

Disable Root Login

# Edit SSH config
sudo nano /etc/ssh/sshd_config

# Set these values:
# PermitRootLogin no
# PasswordAuthentication no

# Restart SSH
sudo systemctl restart sshd

Install Fail2ban

sudo apt install -y fail2ban
sudo systemctl enable fail2ban
sudo systemctl start fail2ban

Fail2ban automatically blocks IPs that attempt brute-force SSH logins.

Keep the Firewall Tight

Only open the ports you need:

  • Port 22 (SSH) ‚Äî Required
  • Port 3000 (OpenClaw Web UI) ‚Äî Or use Tailscale instead
  • Port 80/443 (HTTP/HTTPS) ‚Äî Only if using Nginx with a domain

Monitoring and Maintenance

Once your openclaw vps deployment is running, keep it healthy:

Check Gateway Status

# Check if OpenClaw is running
openclaw gateway status

# View recent logs
journalctl -u openclaw -n 50 --no-pager

Monitor Resource Usage

# Quick system overview
htop

# Disk usage
df -h

Update OpenClaw

# Update to the latest version
sudo npm install -g openclaw@latest

# Restart the daemon
openclaw gateway restart
