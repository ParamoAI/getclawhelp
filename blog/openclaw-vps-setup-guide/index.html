How to Set Up OpenClaw on a VPS: Complete 2026 Guide

Running OpenClaw on a VPS means your AI assistant is always online — 24/7, no laptop required. This guide walks you through every step: picking a VPS provider, installing OpenClaw, configuring your first skills, and connecting your integrations.

Why Run OpenClaw on a VPS?

A Virtual Private Server gives your OpenClaw instance a permanent, always-on home. Unlike running it on your laptop, a VPS stays online even when your computer is off. Your automations trigger on schedule, messages get answered instantly, and your data stays on a server you control.

Key advantages:

  • 24/7 uptime — your assistant never sleeps
  • Data privacy — everything stays on your server
  • Full control — install any skill, any integration
  • Low cost — VPS hosting starts at $5-10/month

Step 1: Choose Your VPS Provider

OpenClaw runs great on any Linux VPS. Here are the most popular options in 2026:

Recommended Providers

  • Hetzner — Best value. Their CX22 plan ($4.85/mo) gives you 2 vCPU, 4GB RAM, 40GB SSD. More than enough for OpenClaw.
  • DigitalOcean — Developer-friendly. The $6/mo droplet (1 vCPU, 1GB RAM) works for basic setups; upgrade to $12/mo for heavier workloads.
  • Contabo — Maximum resources for the price. Their VPS S plan gives you 4 vCPU and 8GB RAM for under $7/mo.
  • Vultr — Good global coverage. The $6/mo plan is solid for most use cases.

Minimum Requirements

  • 1 vCPU (2+ recommended)
  • 1GB RAM (2GB+ recommended)
  • 20GB storage
  • Ubuntu 22.04 or 24.04 LTS

Step 2: Initial Server Setup

Once your VPS is provisioned, SSH into it:

ssh root@your-server-ip

Update the system and create a non-root user:

apt update && apt upgrade -y
adduser openclaw
usermod -aG sudo openclaw
su - openclaw

Install Node.js

OpenClaw requires Node.js 20 or later:

curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
node --version  # Should show v20.x or later

Step 3: Install OpenClaw

With Node.js installed, grab OpenClaw:

npm install -g openclaw
openclaw init

The init command walks you through initial configuration — setting your AI provider API key, choosing your default model, and configuring your first channel (Slack, Discord, Telegram, etc.).

Step 4: Configure Your First Skills

Skills are what make OpenClaw useful. They're modular capabilities you can add:

openclaw skill install web-search
openclaw skill install calendar
openclaw skill install email-assistant
openclaw skill install file-manager

Each skill comes with its own configuration. The setup wizard guides you through connecting APIs and setting preferences.

Step 5: Set Up Process Management

You want OpenClaw to restart automatically if it crashes or if your server reboots:

sudo npm install -g pm2
pm2 start openclaw -- start
pm2 save
pm2 startup

Now OpenClaw will survive reboots and auto-restart on crashes.

Step 6: Secure Your Server

Basic security hardening:

  • Set up SSH key authentication and disable password login
  • Configure a firewall with ufw
  • Enable automatic security updates
  • Consider setting up a reverse proxy with Nginx + SSL
sudo ufw allow OpenSSH
sudo ufw enable
sudo apt install unattended-upgrades -y

Step 7: Connect Your Integrations

The real power of OpenClaw comes from connecting it to your tools:

  • Messaging — Slack, Discord, Telegram, WhatsApp
  • Calendar — Google Calendar, Outlook
  • Email — Gmail, IMAP/SMTP
  • Productivity — Notion, Linear, GitHub
  • Custom — Any API via custom skills

Each integration is configured through OpenClaw's config file or the setup wizard.

Common Issues & Troubleshooting

Out of memory

If you're on a 1GB VPS and seeing crashes, add swap space:

sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

Port conflicts

If another service is using the same port, change OpenClaw's port in the config or stop the conflicting service.

API rate limits

If you're hitting rate limits from your AI provider, consider using multiple API keys or switching to a provider with higher limits.
