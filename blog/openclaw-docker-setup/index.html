OpenClaw Docker Setup: Step-by-Step Deployment Guide

Docker is the gold standard for deploying server applications cleanly and repeatably. If you want a portable, version-controlled, easy-to-update openclaw docker setup, this guide walks you through every step ‚Äî from writing your Dockerfile to running a production-ready OpenClaw instance with docker-compose.

By the end, you'll have OpenClaw running in a Docker container with persistent data, automatic restarts, and connected messaging channels ‚Äî ready for production use.

Why Use Docker for OpenClaw?

You can install OpenClaw directly on a server (bare metal), and it works great that way. So why bother with Docker? Here's what a containerized openclaw docker setup gives you:

üì¶ Isolation

OpenClaw runs in its own container, separate from your host system. No dependency conflicts, no polluting your server's global npm.

üîÑ Easy Updates

Update by rebuilding the image. Roll back by switching to the previous image. No worrying about breaking your server.

üìã Reproducibility

Your entire setup is defined in a Dockerfile and docker-compose.yml. Version control it, share it, deploy it anywhere identically.

üöö Portability

Move from one server to another by copying your compose file and data volume. Works on any Linux host, any cloud provider.

Prerequisites

Before starting your openclaw docker setup, make sure you have:

  • A server or VPS ‚Äî Ubuntu 22.04+ recommended. Even a $5/month VPS works. See our VPS deployment guide for provider recommendations.
  • Docker installed ‚Äî Version 24+ with Docker Compose V2
  • An AI model API key ‚Äî From Anthropic (Claude) or OpenAI (GPT-4)
  • Basic terminal skills ‚Äî You'll be editing files and running commands

Installing Docker (if needed)

If Docker isn't installed on your server yet, here's the quickest method for Ubuntu/Debian:

# Install Docker using the official convenience script
curl -fsSL https://get.docker.com | sh

# Add your user to the docker group (no sudo needed for docker commands)
sudo usermod -aG docker $USER

# Log out and back in, then verify
docker --version
docker compose version

Step 1: Create Your Project Directory

Start by creating a dedicated directory for your OpenClaw Docker deployment:

mkdir -p ~/openclaw-docker
cd ~/openclaw-docker

This directory will hold your Dockerfile, docker-compose configuration, and any workspace files.

Step 2: Write the Dockerfile

Create a Dockerfile that builds an OpenClaw-ready image:

# Dockerfile
FROM node:22-slim

# Install useful utilities
RUN apt-get update && apt-get install -y \
    curl \
    git \
    && rm -rf /var/lib/apt/lists/*

# Install OpenClaw globally
RUN npm install -g openclaw@latest

# Create workspace directory
RUN mkdir -p /home/openclaw/workspace
WORKDIR /home/openclaw/workspace

# Expose the Web Control UI port
EXPOSE 3000

# Start the Gateway
CMD ["openclaw", "gateway", "start", "--foreground"]

Key decisions in this Dockerfile:

  • node:22-slim ‚Äî Lightweight base image with Node.js 22 pre-installed
  • openclaw@latest ‚Äî Installs the latest version; pin a specific version for production stability
  • --foreground ‚Äî Runs the Gateway in the foreground so Docker manages the process lifecycle

Step 3: Create the Docker Compose File

Docker Compose makes managing your container, volumes, and environment variables easy. Create a docker-compose.yml:

# docker-compose.yml
version: '3.8'

services:
  openclaw:
    build: .
    container_name: openclaw-gateway
    restart: unless-stopped
    ports:
      - "3000:3000"
    volumes:
      - openclaw-config:/root/.openclaw
      - openclaw-workspace:/home/openclaw/workspace
    environment:
      - NODE_ENV=production
    env_file:
      - .env
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 15s

volumes:
  openclaw-config:
  openclaw-workspace:

What this does:

  • restart: unless-stopped ‚Äî Container restarts automatically on crash or server reboot
  • volumes ‚Äî Persist OpenClaw's configuration and workspace data outside the container so updates don't wipe your data
  • env_file ‚Äî Loads environment variables from a .env file (your API keys)
  • healthcheck ‚Äî Docker monitors the Gateway and restarts it if it becomes unresponsive

Step 4: Configure Environment Variables

Create a .env file for your API keys and configuration. Never commit this file to version control.

# .env
ANTHROPIC_API_KEY=sk-ant-your-key-here
# Or for OpenAI:
# OPENAI_API_KEY=sk-your-key-here

Add .env to your .gitignore immediately:

echo ".env" >> .gitignore

Step 5: Build and Start

With everything in place, build your image and start the container:

# Build the Docker image
docker compose build

# Start OpenClaw in the background
docker compose up -d

# Check that it's running
docker compose ps

# View the logs
docker compose logs -f openclaw

You should see the OpenClaw Gateway starting up in the logs. Once it's ready, the Web Control UI is available at http://your-server-ip:3000.

Step 6: Run the Onboarding Wizard

For initial configuration, you'll need to run the OpenClaw onboarding wizard inside the container:

# Open an interactive shell in the running container
docker compose exec openclaw bash

# Run the onboarding wizard
openclaw onboard

# Exit the container shell when done
exit

The wizard will guide you through selecting your AI model, entering your API key, and configuring your agent. Since the configuration is stored in the openclaw-config volume, it persists across container restarts and rebuilds.

Step 7: Connect Messaging Channels

Connect your messaging platforms from inside the container:

# Connect WhatsApp
docker compose exec openclaw openclaw channels login whatsapp

# Connect Slack
docker compose exec openclaw openclaw channels login slack

# Connect Telegram
docker compose exec openclaw openclaw channels login telegram

For WhatsApp, the QR code displays in your terminal ‚Äî scan it with your phone. For Slack and Discord, follow the OAuth flow or paste your bot token as prompted.

Channel credentials are stored in the config volume, so they survive container restarts. You won't need to re-scan the WhatsApp QR code every time you update.

Managing Your Docker Deployment

Viewing Logs

# Follow logs in real time
docker compose logs -f openclaw

# View the last 100 lines
docker compose logs --tail 100 openclaw

Stopping and Starting

# Stop OpenClaw (preserves data)
docker compose stop

# Start it back up
docker compose start

# Full restart
docker compose restart

Updating OpenClaw

When a new version of OpenClaw is released, updating your openclaw docker setup is straightforward:

# Rebuild the image with the latest version
docker compose build --no-cache

# Restart with the new image
docker compose up -d

# Verify the new version
docker compose exec openclaw openclaw --version

Your configuration and workspace data persist in Docker volumes, so updates are seamless. If something goes wrong, you can rebuild with a pinned version:

# In your Dockerfile, pin a specific version:
RUN npm install -g openclaw@1.2.3

Backing Up Your Data

Docker volumes hold your configuration and workspace. Back them up regularly:

# Create a backup of OpenClaw volumes
docker run --rm \
  -v openclaw-docker_openclaw-config:/data/config \
  -v openclaw-docker_openclaw-workspace:/data/workspace \
  -v $(pwd)/backups:/backup \
  alpine tar czf /backup/openclaw-backup-$(date +%Y%m%d).tar.gz -C /data .

To restore from a backup, reverse the process ‚Äî extract the archive into the volumes.

Running Multiple Agents

You can run multiple OpenClaw instances in separate containers ‚Äî each with its own configuration, channels, and workspace. Just duplicate the service in your docker-compose.yml with different names and ports:

services:
  openclaw-main:
    build: .
    container_name: openclaw-main
    restart: unless-stopped
    ports:
      - "3000:3000"
    volumes:
      - main-config:/root/.openclaw
      - main-workspace:/home/openclaw/workspace
    env_file: .env

  openclaw-support:
    build: .
    container_name: openclaw-support
    restart: unless-stopped
    ports:
      - "3001:3000"
    volumes:
      - support-config:/root/.openclaw
      - support-workspace:/home/openclaw/workspace
    env_file: .env-support

Adding a Reverse Proxy (Nginx)

For production deployments with HTTPS, add Nginx as a reverse proxy in your compose stack:

services:
  openclaw:
    build: .
    container_name: openclaw-gateway
    restart: unless-stopped
    expose:
      - "3000"
    volumes:
      - openclaw-config:/root/.openclaw
      - openclaw-workspace:/home/openclaw/workspace
    env_file: .env

  nginx:
    image: nginx:alpine
    container_name: openclaw-nginx
    restart: unless-stopped
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/conf.d/default.conf
      - ./certs:/etc/nginx/certs
    depends_on:
      - openclaw

This setup lets you serve the OpenClaw Web UI behind HTTPS with a proper domain name.

Docker vs. Bare Metal: When to Use Each

Both approaches work well for OpenClaw. Here's how to decide:

  • Choose Docker if: You manage multiple services on the same server, want clean updates and rollbacks, need reproducible deployments across environments, or are deploying for clients.
  • Choose bare metal if: You're running OpenClaw on a dedicated machine (like a Mac Mini), want the simplest possible setup, or need maximum performance with zero overhead.

For most production deployments on a VPS, Docker is the recommended approach. For personal use on dedicated hardware, bare metal is simpler.

Troubleshooting

Container Keeps Restarting

Check the logs for errors:

docker compose logs --tail 50 openclaw

Common causes: missing API key in .env, invalid configuration, or port 3000 already in use on the host.

Can't Access Web UI

Verify the container is running and the port is mapped:

docker compose ps
# Check that port 3000 is listed under PORTS

If running on a VPS, ensure your firewall allows traffic on port 3000 (or use a reverse proxy on port 80/443).

WhatsApp Session Lost After Rebuild

Make sure your config volume is properly mounted. The WhatsApp session data is stored in /root/.openclaw ‚Äî if that's not in a persistent volume, you'll need to re-scan the QR code after every rebuild.

Out of Disk Space

Docker images and build cache can accumulate. Clean up with:

# Remove unused images and build cache
docker system prune -a

Frequently Asked Questions

Can I run OpenClaw in Docker?

Yes. OpenClaw runs well in Docker containers. You can use the official Node.js Docker image, install OpenClaw via npm, and manage it with docker-compose for easy deployment, updates, and portability.

What are the benefits of running OpenClaw in Docker vs bare metal?

Docker gives you isolated environments, easy rollbacks, portable configurations, and simpler updates. Bare metal is slightly lower overhead and simpler for single-purpose machines. For most users, Docker is the better choice for production deployments.

How much RAM does OpenClaw need in Docker?

OpenClaw itself typically uses 200‚Äì500MB of RAM. Allocate at least 1GB for the container to allow headroom for skills, browser automation, and spikes. A 2GB container handles most business workloads comfortably.

Can I use docker-compose with OpenClaw?

Yes, docker-compose is the recommended way to manage OpenClaw in Docker. It lets you define your configuration, volumes, and environment variables in a single YAML file for easy management and reproducibility.

How do I update OpenClaw in Docker?

Rebuild your Docker image with the latest OpenClaw version (npm install -g openclaw@latest in the Dockerfile), then run docker compose up -d --build. Your data persists in mounted volumes, so updates are seamless.
